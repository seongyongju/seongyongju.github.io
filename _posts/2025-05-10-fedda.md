## Introduction

Federated Learning은 raw data를 중앙 서버로 모으지 않고 여러 client가 협력해 global model을 학습하는 방식입니다. 이 구조는 privacy와 data governance 측면에서 장점이 있지만, 실제 client data는 대부분 IID가 아닙니다. 각 client가 가진 class 분포, feature 분포, sample 수가 다르기 때문에 local update가 서로 다른 방향으로 움직이고, 단순 averaging을 하면 global model이 불안정해질 수 있습니다.

FedDA가 다루는 핵심 문제는 class imbalance가 있는 Non-IID 상황입니다. 어떤 client는 특정 class 데이터를 충분히 가지고 있지만, 다른 client는 해당 class가 거의 없을 수 있습니다. 이런 상태에서 local training을 하면 각 client model은 자신이 가진 class에 과적합되고, global aggregation 이후에도 tail class 성능이 낮게 유지됩니다.

FedDA는 data augmentation을 이용해 각 client에 부족한 class sample을 보완하고, 동시에 communication overhead를 줄이려는 방법입니다. 단순히 서버가 모든 데이터를 모아 augmentation하는 방식이 아니라, federated setting의 privacy constraint와 communication constraint를 고려해 synthetic sample과 knowledge sharing을 결합합니다.

## Framework

### Conditional AutoEncoder 기반 Augmentation

FedDA는 CVAE(Conditional Variational AutoEncoder)를 사용하여 Non-IID client에 부족한 sample을 생성합니다. CVAE는 class condition을 입력으로 받아 해당 class에 해당하는 feature 또는 sample을 생성할 수 있습니다. 이를 통해 특정 client가 보유하지 않거나 매우 적게 보유한 class에 대해 synthetic data를 만들 수 있습니다.

이 접근의 목적은 local dataset을 완전히 IID로 만드는 것이 아니라, local update가 특정 class에만 치우치는 현상을 완화하는 것입니다. 각 client가 모든 class를 충분히 갖고 있지 않더라도, 생성된 sample이 decision boundary 학습에 도움을 주면 global model의 class-wise 성능 불균형을 줄일 수 있습니다.

Federated setting에서 data augmentation은 일반 supervised learning보다 어렵습니다. Raw data를 중앙에 모을 수 없고, client별 data distribution도 다릅니다. FedDA는 CVAE를 활용해 이 제약 안에서 부족한 class signal을 보강하려고 합니다.

### Knowledge Distillation

FedDA는 모델 parameter나 gradient를 무작정 많이 공유하는 대신 hidden-layer feature 기반 knowledge를 활용합니다. Knowledge distillation은 teacher가 가진 정보를 student가 더 작은 형태로 학습하도록 만드는 방법인데, federated setting에서는 client 간 또는 server-client 간에 raw data 대신 intermediate knowledge를 교환하는 데 활용될 수 있습니다.

이 방식은 communication overhead를 줄이는 데 목적이 있습니다. FL에서는 매 round마다 model parameter를 주고받는 비용이 크고, client 수가 많아질수록 전체 통신량이 문제가 됩니다. FedDA는 augmentation과 distillation을 결합하여 성능 개선에 필요한 정보만 더 효율적으로 전달하려고 합니다.

또한 raw data privacy 관점에서도 의미가 있습니다. 물론 feature나 synthetic data도 privacy leakage 가능성이 완전히 없는 것은 아니지만, 원본 이미지를 직접 공유하는 것보다는 federated learning의 기본 가정에 더 잘 맞습니다.

## Experiments

논문은 Fashion-MNIST와 CIFAR-10에서 다양한 Non-IID 분포를 구성하여 FedDA를 평가합니다. 평가에서는 client별 class distribution이 불균형한 상황을 만들고, FedAvg 계열 baseline과 비교해 global model accuracy가 얼마나 개선되는지 확인합니다.

중요한 관찰은 Non-IID가 심해질수록 단순 aggregation만으로는 성능이 크게 떨어진다는 점입니다. Client가 서로 다른 class subset만 가지고 있으면 각 local model은 자신이 본 데이터에 맞춰 업데이트되고, 이 update들을 평균내면 global model이 안정적인 representation을 만들기 어렵습니다. FedDA는 synthetic data로 부족한 class signal을 보강하여 이 문제를 완화합니다.

또한 communication cost를 함께 평가합니다. Federated learning 연구에서 accuracy만 높이는 방법은 많지만, 통신량이 너무 크면 실제 distributed environment에서 사용하기 어렵습니다. FedDA는 성능 개선과 통신 효율을 함께 고려한다는 점에서 실용적인 의미가 있습니다.

## Conclusion

FedDA는 Non-IID 문제를 data augmentation과 communication efficiency 관점에서 함께 다룹니다. Federated setting에서는 성능뿐 아니라 어떤 정보를 얼마나 자주 교환하는지도 중요하므로, knowledge distillation과 synthetic sample generation의 결합이 의미 있습니다.

제가 이 논문에서 중요하게 본 부분은 Non-IID 문제를 optimization algorithm만으로 해결하려 하지 않고, data distribution 자체를 보강하려고 한 점입니다. 실제 FL 환경에서는 client가 가진 데이터가 제한적이고 편향되어 있기 때문에, local training data를 어떻게 보완할지가 중요합니다. FedDA는 synthetic data가 federated learning의 class imbalance 문제를 완화하는 도구가 될 수 있음을 보여줍니다.
