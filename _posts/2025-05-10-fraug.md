## Introduction

Federated Learning에서 Non-IID 문제는 보통 label distribution shift를 중심으로 다뤄졌습니다. 예를 들어 어떤 client는 class A 데이터를 많이 가지고 있고, 다른 client는 class B 데이터를 많이 가지고 있는 상황입니다. 하지만 실제 환경에서는 label distribution이 비슷하더라도 client별 feature distribution이 달라지는 경우가 많습니다. 같은 class의 이미지라도 병원, 카메라, 센서, 지역, 촬영 조건에 따라 feature가 달라질 수 있습니다.

이런 feature distribution shift는 단순 label skew보다 더 미묘합니다. 겉으로는 모든 client가 같은 class label을 가지고 있어 균형 잡힌 것처럼 보이지만, 각 client model은 서로 다른 feature pattern을 학습합니다. Aggregation 단계에서 이런 update를 평균내면 global representation이 어느 한 domain에도 안정적으로 맞지 않거나, 특정 client domain에 편향될 수 있습니다.

FRAug는 feature-space Non-IID 문제를 representation augmentation으로 완화하려는 방법입니다. Raw input image를 직접 생성하기보다, learned representation space에서 client 간 feature gap을 줄이는 방향으로 augmentation을 수행합니다.

## Framework

### Feature Distribution Shift

Client들이 같은 label을 가지고 있어도 촬영 환경, domain, sensor, style이 다르면 feature representation이 서로 달라질 수 있습니다. 예를 들어 medical image federated learning에서 병원마다 scanner 종류나 imaging protocol이 다르면 같은 질병 label이라도 image texture와 intensity distribution이 다릅니다. 이 경우 모델은 label 자체보다 client-specific feature에 의존하게 될 수 있습니다.

Feature distribution shift가 어려운 이유는 server가 raw data를 볼 수 없다는 점입니다. 중앙 서버가 모든 client data를 모아 domain alignment를 수행할 수 있다면 문제를 직접 분석할 수 있지만, FL에서는 privacy constraint 때문에 각 client의 feature distribution을 제한적으로만 파악할 수 있습니다.

FRAug는 이런 상황에서 representation level의 augmentation을 사용합니다. Client가 학습한 representation을 더 다양한 feature variation에 노출시켜, local model이 자신의 domain에만 과적합되는 것을 줄입니다.

### Federated Representation Augmentation

FRAug는 shared embedding generator를 학습하여 client consensus를 반영한 synthetic embedding을 생성합니다. 이 generator는 여러 client의 representation 정보를 반영하여, 특정 client에만 치우치지 않는 feature variation을 만드는 역할을 합니다. 이후 client-specific transformation을 통해 각 client의 training space를 보강합니다.

이 접근은 input space augmentation과 다릅니다. 이미지 자체를 생성하려면 high-quality generative model이 필요하고, privacy leakage나 생성 품질 문제가 생길 수 있습니다. Representation augmentation은 이미 model이 학습한 feature space에서 variation을 추가하기 때문에, 상대적으로 task-relevant한 변화를 만들기 쉽습니다.

또한 FRAug는 feature skew를 직접 겨냥합니다. Label imbalance를 해결하는 방법이 class frequency를 맞추는 데 집중한다면, FRAug는 같은 label 안에서도 client마다 다른 feature mode를 더 잘 포괄하도록 학습을 유도합니다.

## Experiments

논문은 public benchmark와 real-world medical dataset에서 FRAug를 평가합니다. Public benchmark에서는 controlled setting으로 feature shift를 구성하고, medical dataset에서는 실제 기관 간 domain shift가 존재하는 상황을 다룹니다. 이러한 평가 구성이 중요한 이유는 feature Non-IID가 실험적으로 만든 문제뿐 아니라 실제 FL deployment에서도 자주 발생하기 때문입니다.

결과적으로 FRAug는 feature distribution shift가 있는 상황에서 기존 FL method보다 global model 성능을 개선할 수 있음을 보입니다. 특히 client domain이 서로 다를수록 representation augmentation이 local overfitting을 줄이고, global model이 더 일반적인 feature를 학습하도록 돕습니다.

이 논문은 Non-IID 평가에서 label skew만 보는 것이 부족하다는 점도 보여줍니다. 실제 application에서는 label 분포가 어느 정도 균형적이어도 domain-specific feature 때문에 global performance가 낮아질 수 있습니다.

## Conclusion

FRAug는 Non-IID 문제를 label skew만이 아니라 feature skew 관점에서 다뤘다는 점이 중요합니다. 실제 federated setting에서는 client마다 sensing 환경과 domain이 다를 수 있으므로, representation-level augmentation은 실용적인 접근입니다.

제가 이 논문에서 중요하게 본 부분은 데이터 불균형을 단순 class count 문제가 아니라 representation 문제로 본 점입니다. SAFE AI나 Federated Learning에서 Non-IID를 해결하려면 client별 label 분포뿐 아니라 feature distribution과 domain shift까지 함께 봐야 합니다. FRAug는 그 문제를 augmentation 관점에서 풀어낸 사례입니다.
